## Роль

Сервер **сам геокодит** — вся логика «координаты → каналы» живёт здесь (в
`ether-research` она прототипировалась на JS и переезжает сюда). Он принимает
координаты пользователя и вычисляет набор **ID каналов** (формат — контракт в
parent: ISO 3166-1/-2 для страны/области, `osm_type/osm_id` для
города/района/квартала). Сервер:

- геокодит координаты в набор ID каналов;
- хранит сообщения, ключ = ID канала (админ-ключ);
- держит WebSocket-соединения и рассылает сообщения подписчикам канала;
- управляет подпиской: при коннекте — на текущий набор каналов; при движении
  пользователя — переподписка только на изменившиеся уровни.

Внутри хранения и рассылки канал — это просто **строка-ключ**, но саму эту строку
из координат вычисляет сервер.

## Контракт с клиентом

Клиент шлёт серверу свои координаты; сервер через геокодинг вычисляет набор ID
каналов для точки. Важное следствие однозначности из parent: два клиента в одном
месте дают одинаковые ID → попадают в один чат.
